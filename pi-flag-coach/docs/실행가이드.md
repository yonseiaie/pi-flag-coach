# 제출 가이드 — GitHub 올리기 & Pi 사용법

## A. GitHub에 올리기 (Windows PowerShell)

> 이 PowerShell은 `&&` 를 못 씁니다. **한 줄씩** 복붙해서 엔터하세요.

**0) 압축을 풀고 그 폴더로 이동** (아직이라면)
```powershell
cd ~\Downloads
Expand-Archive pi-flag-coach.zip -DestinationPath .
cd pi-flag-coach
```
폴더가 맞는지 확인 — `ls` 했을 때 `README.md`, `web`, `skills`, `extensions` 가 보이면 OK.

**1) (이 PC에서 git 처음이면) 이름/이메일 설정**
```powershell
git config --global user.name "본인이름"
git config --global user.email "본인이메일"
```

**2) 커밋 (전부 로컬, 인터넷 불필요)**
```powershell
git init
git add .
git commit -m "pi-flag-coach: 플래그풋볼 작전 보드 & 코칭 에이전트"
```

**3) GitHub에서 빈 저장소 먼저 생성**
github.com → New repository → 이름 `pi-flag-coach` → **Public** →
README/.gitignore/License 추가 옵션은 **체크하지 말 것**(이미 폴더에 있어 충돌남) → Create.

**4) 원격 연결 후 업로드** (`<본인아이디>` 를 실제 아이디로 교체)
```powershell
git remote add origin https://github.com/<본인아이디>/pi-flag-coach.git
git branch -M main
git push -u origin main
```
로그인 창이 뜨면 GitHub 계정으로 로그인. 끝나면 저장소 링크를 제출.

> push 전에 `README.md` 와 `docs/보고서.md` 안의 `<your-id>` / `<본인아이디>` 를 실제 아이디로 바꿔두세요.

## B. Pi 사용법 & "Pi에게 시킬 프롬프트"

### 1) 설치 & 인증
```powershell
irm https://pi.dev/install.ps1 | iex
$env:ANTHROPIC_API_KEY="sk-ant-..."     # 또는  pi  실행 후  /login
```

### 2) 웹 UI를 Pi와 연결해 실행
```powershell
npm install
cd extensions ; npm install ; cd ..
npm run web                              # http://localhost:5173
```
보드에서 작전을 그리고 **"🤖 이 작전 분석시키기"** 를 누르면 Pi가 평가합니다.

### 3) Pi 대화형으로 직접 시키기 (발표 영상용 — "Pi로 만들/쓴 증거"가 세션에 남음)
```powershell
npm run agent
```
열린 Pi 창에 아래 프롬프트를 그대로 붙여 시연하세요.

**(a) 작전 분석 — play-review 스킬 + 도구 사용 보여주기**
```
/skill:play-review 다음 작전을 평가해줘:
{"name":"Cross","formation":"SPREAD","players":[
 {"id":"X","preset":"cross","routeYards":11},{"id":"Y","preset":"cross","routeYards":12},
 {"id":"Z","preset":"go","routeYards":19},{"id":"C","preset":""},{"id":"QB","preset":"rollout"}]}
```

**(b) 상대 수비 실시간 분석 — analyze-defense + read_opponent_record 도구**
```
read_opponent_record 도구로 상대 기록을 불러오고, analyze-defense 임계값으로 다음 플레이를 추천해줘.
```

**(c) 선수 라인업 조회 — player-stats 스킬**
```
/skill:player-stats X와 Z 포지션 선수의 역할과 스탯을 알려줘.
```

**(d) 훈련 계획 자동 생성 — training-planner 스킬 + create_training_session 도구**
```
/skill:training-planner 이번 주 토요일 훈련 계획을 60분짜리로 만들어서 저장해줘.
목표는 딥 패스 정확도와 블리츠 대응이야.
```

**(e) 멀티 에이전트 — 서브에이전트 병렬 위임**
```
defense-analyst 와 offense-strategist 서브에이전트를 써서 상대 라이벌즈에 대한
수비 약점 분석과 그에 맞는 공격 콜을 동시에 만들어줘.
```

### 4) Pi에게 "기능 확장"을 시키는 프롬프트 (코딩 에이전트로 직접 만들게 하기)
아래처럼 시키면 Pi가 코드를 직접 수정/추가하고, 그 과정이 세션 기록으로 남습니다.
이게 "Pi로 만들었다"의 가장 확실한 증거입니다.
```
이 저장소 구조를 보고, web/index.html 의 프리셋 라우트에 'wheel' 과 'dig' 두 개를 추가해줘.
applyPreset 의 tbl 객체에 좌표를 넣고, <select id="preset"> 에 옵션도 추가해.
```
```
extensions/flag-tools.ts 에 'compare_players' 라는 도구를 새로 등록해줘.
두 포지션 id를 받아 data/roster.json 에서 스탯을 비교해 표로 반환하도록.
```

> 세션 기록 위치: `~/.pi/agent/sessions/` — 발표 영상에서 이 기록을 보여주면 Pi 활용 증빙이 됩니다.
