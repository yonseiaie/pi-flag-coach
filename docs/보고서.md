# 기말 프로젝트 보고서 — pi-flag-coach

플래그풋볼 실시간 전술 코칭 & 팀 관리 AI 에이전트

---

## 1. 선택한 시나리오

**실시간 전술 감시 및 피드백 + 맞춤형 팀 관리 솔루션.**

특정 데이터 소스를 감시하다 중요한 변화가 생기면 알림을 주는 Pi Agent의 특성을 활용해,
상대 팀의 수비 전술 변화를 실시간으로 분석하고 최적의 플레이 호출·전술 수정을 제안한다.
동시에 팀 일정·훈련 세션·선수 활동을 관리하는 지능형 비서를 하나의 웹 인터페이스로 제공한다.

## 2. 문제 정의

플래그풋볼 코치는 경기/연습 중 짧은 시간 안에 의사결정을 내려야 하지만, 근거가 되는 통계
(상대 세이프티 깊이, 플랫 수비 성공률, 블리츠 빈도, 자기 팀 선수 스탯)는 흩어져 있고 수작업
조회가 느리다. 또한 일반 챗봇에 질문하면 (1) 데이터를 사람이 복사해 넣어야 하고 (2) 답변 형식이
매번 달라 재현이 안 되며 (3) 근거 없는 환각 위험이 있고 (4) 외부 시스템 기록이 안 된다.

→ **데이터를 스스로 읽고, 절차를 지키며, 외부 DB와 연동하고, 결과를 추적 가능하게 남기는
에이전트형 코칭 도구**가 필요하다.

## 3. 서비스 대상 사용자

- **헤드코치/코디네이터** — 경기 중 플레이 콜 결정, 전술 수정.
- **선수 관리 담당** — 선수 스탯 추적, 약점 기반 훈련 설계.
- 아마추어~세미프로 플래그풋볼 팀(소규모, 전담 분석관이 없는 환경)을 1차 대상으로 한다.

## 4. 핵심 기능

5인제 플래그풋볼(X·Y·Z·C·QB) 기준의 **작동하는 작전 보드**가 중심이다.

1. 선수 5명 드래그 배치, 라우트 직접 그리기(자유 곡선) + 프리셋(go/slant/out/in/cross/curl/comeback/flat/hitch).
2. 재생: 그린 라우트대로 선수가 움직이는 애니메이션으로 작전 검증.
3. 작전 저장/불러오기(이름별) + JSON 내보내기/가져오기로 플레이북 관리.
4. 작전 설명 자동 생성(라우트→야드·전개 방향 텍스트).
5. Pi 분석/평가: 그린 작전 JSON을 에이전트에 보내 강점·약점·보완점 평가.
6. 보조 코칭: 수행 역할 조회, 상대 수비 지표 모니터링, 훈련 세션 자동 생성, 멀티 에이전트 위임.

## 5. 시스템 구조

```
웹 대시보드(web/index.html)
   │ POST /api/chat  ── 사용자 프롬프트가 에이전트를 깨움(triggerTurn 패턴)
   ▼
브리지 서버(server/bridge.mjs, Express) ── spawn pi -p
   ▼
Pi 에이전트 코어(루프·세션·LLM 호출) ── AGENTS.md 프로젝트 지침
   ├─ Skills      : player-stats · analyze-defense · training-planner
   ├─ Extensions  : flag-tools(커스텀 도구) · tactical-monitor(실시간 컨텍스트 주입+위젯)
   ├─ MCP         : stats-db(sqlite→postgres) · team-store(filesystem)
   └─ Subagents   : defense-analyst ∥ offense-strategist
   ▼
응답 → 브리지 → 웹 UI(전략 힌트·코칭 리포트 갱신)
```

데이터 계층은 `data/`(roster.json, opponents.json, plays.json)이며, MCP를 통해 통계 DB로
승격할 수 있다. 산출물(훈련 계획·리포트)은 `.gitignore` 처리해 저장소를 깨끗이 유지한다.

## 6. Skill / MCP / Pi Extension 활용 방식

### Skill (`skills/<name>/SKILL.md`)
프론트매터의 `name`/`description`으로 LLM이 **필요할 때만** 로드한다(progressive disclosure).

- **player-stats**: 선수/포지션 질문 시 로드. 세부 매치업은 `references/positions.md`로 분리해 추가 토큰을 아낀다.
- **analyze-defense**: 수치 임계값(세이프티≥70 / 플랫≤50 / 블리츠≥65)으로 추천을 결정. 추측 금지·근거 강제.
- **training-planner**: 약점→드릴 매핑으로 세션 구성 후 도구로 저장.

### Pi Extension (`extensions/*.ts`)
`export default function(pi: ExtensionAPI)` 팩토리에서 도구/이벤트를 등록한다.

- **flag-tools.ts**: `pi.registerTool` + typebox 스키마로 `list_players`/`get_player_stats`/
  `read_opponent_record`/`save_play`/`create_training_session` 5개 도구를 등록. 도구가 직접
  `data/*.json`을 읽고 써서 "팀 관리"를 수행한다.
- **tactical-monitor.ts**: `session_start`에서 감시 위젯을 켜고, **`before_agent_start`에서 매 턴
  직전 현재 수비 지표를 메시지로 주입**해 "실시간 감시"를 구현한다. `coaching_report` 도구와
  `/notify` 백그라운드 알림도 제공한다.

### MCP (`.mcp.json`)
무인증으로 즉시 도는 공식 서버(`server-sqlite`, `server-filesystem`)를 기본값으로 두어 채점/시연이
바로 가능하게 했고, 운영 전환 시 `server-postgres`로 정의만 교체하면 되도록 `_production_swap`을
문서화했다. 에이전트는 MCP로 경기 통계 DB를 직접 읽어 상대 기록 차트의 근거를 확보한다.

## 7. Web UI 설명

`web/index.html` 단일 페이지(빌드 불필요). 어두운 테마, 3분할 레이아웃:

- **좌측**: 내비게이션 + 사용법 안내 카드.
- **중앙(핵심)**: SVG **인터랙티브 작전 보드**. 선수 5명(X·Y·Z·C[정사각형]·QB)을 드래그로 배치하고,
  *그리기* 모드에서 선수를 끌면 자유 곡선 라우트가 그려지며, 우측 *프리셋* 드롭다운으로 정형 라우트를
  자동 생성한다. *재생* 버튼은 `requestAnimationFrame` 으로 각 선수를 라우트 폴리라인을 따라 보간 이동시킨다.
  *지우기/기본 포메이션/전체 지우기* 도구 포함.
- **우측**: 수행 역할(클릭 시 보드에서 해당 선수 선택), 라우트→야드·전개 방향을 **자동 텍스트화한 작전 설명**,
  **Pi 분석/평가** 결과, 작전 **저장/불러오기**(localStorage 슬롯 + JSON 파일).

"이 작전 분석시키기"를 누르면 현재 작전이 JSON으로 직렬화되어 `/api/chat` → 브리지 → Pi 에이전트
(`play-review` 스킬)로 전달되고, 강점/약점/보완 평가가 우측에 표시된다. API 키가 없으면 휴리스틱
데모 분석으로 폴백해 시연이 끊기지 않는다. 페이지를 열면 예시 작전 **"Cross"** 가 미리 로드된다.

## 8. 구현 결과

- Pi 레퍼런스(`pi-work-agent`)와 동일한 디렉터리 규약(extensions·skills·.mcp.json·.pi/agents·MIT)을 따랐다.
- 스킬 3개 / 커스텀 도구 6개 / MCP 서버 2개 / 서브에이전트 2개 / 웹 대시보드 1개를 구현.
- 웹 UI는 제시된 디자인(다크 테마·전술 보드·원형 차트)을 그대로 재현하며, API 키 유무와 무관하게 렌더링된다.
- CLI에서 `--skill` ON/OFF 비교와 `-c` 세션 재개로 스킬 효과·관찰가능성을 검증할 수 있다.

## 9. 한계점 및 개선 방향

| 한계 | 개선 방향 |
|-----|---------|
| 전술 보드가 PLAY 01 루트만 시각화 | plays.json의 routes를 좌표화해 플레이별 자동 렌더링 |
| 브리지가 print 모드(요청당 호출) | pi **RPC/SDK 모드**로 전환해 스트리밍·연속 세션 지원 |
| 실시간 지표가 샘플 데이터 | 라이브 스코어/트래킹 API를 MCP로 연결해 진짜 실시간화 |
| 임계값이 고정 규칙 | 과거 경기 학습 기반 동적 임계값으로 개인화 |
| 보안/권한 | MCP 쓰기 권한을 역할별로 제한, 신뢰 설정 적용 |

## 10. 단순 프롬프트 대비 에이전트의 가치

| 항목 | 단순 프롬프트 | 본 에이전트 |
|-----|------------|-----------|
| 데이터 접근 | 사람이 복사·붙여넣기 | 도구로 스스로 조회 |
| 일관성 | 매번 형식 다름 | 스킬이 형식·규칙 고정 |
| 근거 | 환각 위험 | 수치 임계값·MCP DB 근거 |
| 외부 연동 | 수동 | MCP로 직접 읽기/쓰기 |
| 추적 | 대화 닫으면 소멸 | 세션 JSONL 저장·재개 |
