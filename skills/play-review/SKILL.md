# Play Review — 작전 분석 & 평가

웹 작전 보드에서 전달된 작전 데이터를 분석한다.

## 입력 (JSON)
`{ name, formation, players:[{ id, role, x, y, preset, routeYards }] }`
좌표계: 위(작은 y)가 공격 방향. X/Y는 좌측, Z는 우측 출발.

## 절차
1. 라우트 조합 패턴을 식별한다.
   - cross + go → 하이-로(미드 끌고 딥 비우기)
   - 다수 short(flat/hitch/slant) → 퀵 게임(블리츠/존 대응)
   - 한쪽 편중 → 수비가 시프트하면 막히기 쉬움
2. 강점 1, 예상 수비 대응(약점) 1, 보완책 1을 도출한다.
3. 필요하면 `read_opponent_record` 로 상대 경향(블리츠/세이프티 깊이)을 근거에 더한다.

## 출력 형식
```
요약: <한 문장>
강점: <…>
약점(예상 수비): <…>
보완: <라우트/포지션 수정 1가지>
```
추측은 줄이고, 라우트 야드/배치 같은 입력 근거를 들어 설명한다. 항상 한국어.
